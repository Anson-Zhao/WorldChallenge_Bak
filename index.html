Establishing User-Friendly Access to the Comprehensive Visualization of USGS Data Sets

Fei Tian College and Northern Academy of the Arts,Middletown, NY USA - A World Bridge Project

A Collaborative Model for Acquiring Current and Archived USGS Data and Visualization in NASA WebWorldWind

A World Bridge - Trillium Learning

The United Nations FAO Invasive Species Project is a collaboration between A World Bridge teams at Feitian College and the Northern Academy of the Arts in Middletown, New York, USA.

Goal

The NAA A World Bridge team aims to create a user-friendly platform for the public to access USGS geospatial data, including WMS and WCS-served data.

This can be done with a virtual globe that allows users to search, discover and stack USGS data, using links to native sources. To accomplish this, the students worked with USGS, NASA and other contractors to

▶      find USGS web-served geospatial data (maps and coverages)

▶      create a searchable index tool to find and display these data on a WorldWind impletation

▶      adapt existing WorldWind tools to offer display options on a virtual globe

This project consists of two web applications using NASA’s WebWorldWind geospatial platform and includes place marks, heat maps and drone-retrieved images contained in different layers. These layers can be adjusted according to the user’s objective and options for tilting and zooming in enhance the experience.

Features Overview

Educational Outreach


This NASA WorldWind platform is designed to communicate real-time data collections from USGS to the public in a uniquely visualized yet comprehensive form. Featuring data from the National Map Services and USWTDB, the application improves the public’s understanding of geospatial information.

The UUSGS-NASA Data Visualization Project application is a completely open source application based on NASA-ESA WebWorldWind. Programming code utilized a variety of tools and applications, including Javascript, HTML5, CSS, Python, and server-side programming - all available on the GitHub site.

Data Workflow


The data workflow for this project is described in the graphic below. The process consists of:



1. Initial data collection and information analysis from the USGS website

2. Verification and latitude-longitude processing for each USGS location

3. Output data and/or error to data container

4. Server-side data storage and WebWorldWind layer foundation

5. Data visualization of WWW on the application, GUI functionality and client calls.
